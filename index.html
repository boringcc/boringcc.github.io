MyBatis DAO层实现的两种方法

使用mybatis实现DAO层的2种方式


第一种方法：基于注解的配置方式


通过注解配置的方式要注意普通sql语句和动态sql两种情况。
普通sql的注解有@Select,@Delete,@insert,@update四种，可以完成常见的CRUD操作。
使用方法：以@Select为例通过ID查询信息，其他3种类似。

@Select("select * from "+DEPTTABLE+" where ID = #{id}")
Dept selectById(int id);

要实现复杂的动态sql查询，则需要额外创建Provider类来实现。
支持动态sql的注解有@SelectProvider,@InsertProvider,@UpdateProvider,@DeleteProvider四种
注解中常用的2种参数
①type:指定动态sql方法的类
②method:该类中对应的方法
使用方法：动态查询总数量

@SelectProvider(type=DeptDynaSqlProvider.class,method="count")
Integer count(Map<String, Object> params);

通过该注解会自动匹配到DeptDynaSqlProvider类中的count方法

//动态查询总量
public String count(final Map<String, Object> params){
    return new SQL(){
        {
            SELECT("count(*)");
            FROM(DEPTTABLE);
            if(params.get("dept") != null){
                Dept dept = (Dept) params.get("dept");
                if(dept.getName() != null && !dept.getName().equals("")){
                    WHERE("  name LIKE CONCAT ('%',#{dept.name},'%') ");
                }
            }
        }
    }.toString();
}

以上就是通过注解的方式来实现与数据库的交互。


第二种方法：通过mapper.xml的方法实现DAO层。


mapper.xml可以通过mybatis自动生成，但是会有部分地方要修改。
建议先通过mybatis自动生成，然后自己去修改成想要的状态，自动生成方法为MBG，可以自己去看一下。

这是DAO层的一个接口。

long countByExample(EmployeeExample example);

下面是该DAO层对应的mapper.xml文件中与上面对应的sql查询方法。
xml文件开始要指明对应的类。这样才能找到对应的方法。
mapper.xml中主要是用标签组合成sql语句，具体的使用方法可以自行了解。

<mapper namespace="com.cc.crud.dao.EmployeeMapper">

使用mapper.xml方法可以直接在xml文件中写动态sql语句。
以上就是mybatis实现DAO层的两种方法的大致了解。
